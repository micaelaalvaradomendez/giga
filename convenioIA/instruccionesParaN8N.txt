CONFIGURACIÓN MINIO 

1. ACCESO AL CONTENEDOR MINIO:
   docker exec -it giga-minio /bin/bash

2. CONFIGURAR ALIAS MINIO:
   mc alias set myminio http://localhost:9000 giga-user giga-password-change-me

3. CREAR USUARIO N8N:
   mc admin user add myminio n8n-user n8n12345

4. ASIGNAR PERMISOS AL USUARIO:
   mc admin policy attach myminio readwrite --user n8n-user

5. CREAR BUCKET CONVENIO:
   mc mb myminio/convenio

6. CONFIGURAR ACCESO PÚBLICO:
   mc anonymous set public myminio/convenio
   mc anonymous set download myminio/convenio

7. SUBIR ARCHIVO DEL CONVENIO:
   - Copiar archivo desde host: docker cp "...\giga\convenioIA\convenio.pdf" giga-minio:/tmp/convenio.pdf
   - Subir a bucket: mc cp /tmp/convenio.pdf myminio/convenio/

8. VERIFICAR CONFIGURACIÓN:
   mc ls myminio
   mc ls myminio/convenio

TESTING LOCAL:
Para probar de forma local:
    - Instalar extensión Thunder Client
    - Abrir extensión
    - Hacer new request
    - Cambiar GET a POST
    - Ingresar la URL:
         TEST: http://localhost:5678/webhook-test/3ebdf75e-f0e2-4d18-b070-498f1486d845
    - En body se puede consultar como JSON:
        ej: 
        {
            "chatInput": "Cuantos dias por embarazo corresponden?"
        }
    - Se activa el test en N8N y luego se envía desde VS Code

    -Para prod no necesitan activar nada, solo cambian la URL:
    PROD: http://localhost:5678/webhook/3ebdf75e-f0e2-4d18-b070-498f1486d845
